# Offspace AI Office — 공유 설계 메모

## 목적
대표님이 “지금 일이 진행되고 있나?”를 직접 물어보지 않아도, 대기 대시보드만 보고 현재 작업 상태를 알 수 있게 한다.

## 확정 방향
- 기본 레이아웃: C안 하이브리드 업무관제 대시보드
- 중앙: 픽셀 사무실 / 업무공간
- 좌측: 업무 영역 목록
- 우측: 실시간 에이전트 큐
- 캐릭터: 실제 실행 주체와 1:1 매핑

## 캐릭터 매핑
- 허사장: Hermes 작업
- 코부장: Claude Code 작업
- 오과장: Codex CLI 작업
- 젬대리: Gemini CLI 또는 Antigravity CLI 작업

## 표시 규칙
- idle: 자기 자리에서 대기
- active: 캐릭터가 움직이고 상태등 점멸
- parallel: 같은 캐릭터 분신 생성, `²`, `³` 배지 표시
- blocked: 말풍선에 막힌 이유 표시
- done: 분신이 본체로 합쳐지고 완료 말풍선 표시

## 허사장 / 코부장 역할 분담
- 허사장: 전체 UX, 상태 규칙, 업무 분류, 이벤트 스키마, 대표님 보고
- 코부장: 실제 대시보드 구현, SVG 캐릭터 연결, 애니메이션/상태 렌더링

## 메신저 보고 원칙
- 대표님이 답답하지 않도록 중간중간 짧게 보고한다.
- 보고 형식: `진행중 / 막힘 / 완료 / 다음 작업`만 간단히.
- 대시보드가 완성되면 보고보다 대시보드 자체가 현재 상태의 1차 소스가 된다.
